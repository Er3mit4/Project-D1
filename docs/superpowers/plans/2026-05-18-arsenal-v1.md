# Arsenal v1 Implementation Plan

**Goal:** Build the Arsenal v1 module for D&D 5e PT-BR equipment and magic-item lookup using Livro do Jogador and Guia do Mestre.

**Current extraction decision:** Use Docling, but never against the complete PDFs for this module. The stable path is to split the relevant page ranges first and convert those smaller PDFs with conservative settings.

## Files

- `Arsenal/prepare_docling_sources.py`: creates section PDFs under `Arsenal/docling_out/sources/`.
- `Arsenal/run_docling_extraction.py`: runs Docling with stable per-section settings.
- `Arsenal/extract_arsenal.py`: parses Docling JSON into normalized Arsenal data.
- `Arsenal/validate_arsenal.py`: validates schema plus sentinel items that previously exposed parsing drift.
- `Arsenal/arsenal_raw.json`: normalized extraction envelope with counts and method metadata.
- `Arsenal/arsenal.json`: final app data.
- `Arsenal/template.html`: data-free app template.
- `Arsenal/build_html.py`: injects JSON into the template.
- `Arsenal/index.html`: generated offline app.

## Extraction Workflow

Run from `D:\Python project\Projeto D1` in PowerShell:

```powershell
.venv\Scripts\python.exe Arsenal\run_docling_extraction.py
.venv\Scripts\python.exe Arsenal\extract_arsenal.py
.venv\Scripts\python.exe Arsenal\validate_arsenal.py Arsenal\arsenal.json
.venv\Scripts\python.exe Arsenal\build_html.py
```

Docling settings:

- PHB equipment pages: PDF 145-161, `--no-ocr`, `--page-batch-size 1`, `--num-threads 1`, `--pdf-backend pypdfium2`, `--table-mode accurate`.
- DMG magic item pages: PDF 151-220 and 221-229, `--no-ocr`, `--no-tables`, `--page-batch-size 1`, `--num-threads 1`, `--pdf-backend pypdfium2`.

Root cause of the earlier Docling failure:

- The full PHB conversion with default OCR/table settings failed in preprocessing with `std::bad_alloc`.
- Section PDFs convert successfully with OCR disabled, one-page batches, one thread, and `pypdfium2`.
- PHB tables need `accurate`; `fast` left cells blank in rows such as `Poção de cura`.
- DMG item text needs parser-side ordering by page, column, and vertical position; otherwise two-column pages can hide items such as `Anel de Calor`.

## Validation Anchors

`Arsenal/validate_arsenal.py` must keep checking these sentinel records:

- `Espada Longa`: price `15 po`, damage `1d8 cortante`, weight `1,5 kg`.
- `Placas`: price `1.500 po`, CA `18`, weight `32,5 kg`.
- `Poção de cura`: price `50 po`, weight `0,25 kg`.
- `Anel de Proteção`: rarity `raro`, requires attunement.
- `Anel de Calor`: rarity `incomum`, requires attunement.
- `Aljava de Ehlonna`: rarity `incomum`, no attunement.
